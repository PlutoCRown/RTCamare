# 结构优化

拆分首页、发送页、接受页。确保他们【初始化、错误、进行中】的 3 个状态及功能正确。
必要时候可以拆分文件、拆分文件夹、拆 web-component。
如果拆分 css 记得按页面做好命名前缀。

# 功能优化

1. sender/viewer 服务端报错 exist 时候展示错误状态，并且不要进入进行中的页面
2. sender 退出时，viewer 允许再次等待新 sender，而不是保持当前位置
3. viewer 的 UI 界面，直接全屏开播，而非小窗口
4. viewer 页面的 UI 信息全部绝对定位并支持折叠，允许使用浏览器的新 api 例如 dialog 等。
5. viewer/sender 页面显示推送带宽、分辨率等信息信息
6. sender 推流中的界面 支持选择摄像头
